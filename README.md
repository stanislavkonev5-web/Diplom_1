# Автоматизация тестирования сервиса Stellar Burgers (Часть 1: Unit-тесты)

Проект посвящен обеспечению качества логики заказа в приложении **Stellar Burgers**. На данном этапе реализовано полное покрытие бизнес-логики бэкенда юнит-тестами с использованием современной методологии изоляции зависимостей.

## 🛠 Технологический стек
* **Java 11**
* **JUnit 4** — тестовый фреймворк
* **Mockito** (4.11.0) — библиотека для создания моков и стабов
* **Maven** — управление зависимостями и сборка проекта
* **JaCoCo** — инструмент оценки покрытия кода тестами

## 🎯 Что было реализовано

### 1. Юнит-тестирование класса `Burger`
* **Изоляция компонентов:** Использованы моки и стабы (`Mockito`) для имитации работы классов `Bun` и `Ingredient`. Это позволило протестировать логику сбора бургера в полной изоляции от внешних объектов.
* **Функциональные проверки:** Покрыты тестами методы добавления, удаления и перемещения ингредиентов в чеке.
* **Верификация взаимодействий:** Для метода `getReceipt()` настроена верификация вызовов методов моков, что подтверждает корректную сборку итоговой строки заказа.

### 2. Параметризованное тестирование
* Реализован **Data-Driven подход** для метода `getPrice()`.
* Протестированы различные граничные сценарии:
    * Стандартный расчет стоимости;
    * Обработка заказов без ингредиентов;
    * Работа с дробными значениями цен (`float`).

## 📊 Результаты покрытия (Code Coverage)
Благодаря использованию моков и тщательному подбору тестовых сценариев, достигнуто **100% покрытие кода** для ключевого класса бизнес-логики `Burger`.
